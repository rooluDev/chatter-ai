# 📋 PRD (Product Requirements Document)

## 실시간 채팅 + AI 어시스턴트 플랫폼

> 작성 기준일: 2025년  
> 이 문서는 구현의 기준이 된다. 개발 중 발생하는 모든 의사결정은 이 문서에 근거한다.

---

## 1. 제품 개요

### 1.1 제품명

**ChatterAI** — 실시간 채팅 + AI 어시스턴트 협업 플랫폼

### 1.2 제품 설명

채널 기반의 실시간 채팅 공간에서 팀원 간 소통과 함께, `@AI` 멘션으로 Claude AI 어시스턴트를 호출할 수 있는 협업 플랫폼.  
WebSocket(STOMP) + Redis Pub/Sub 기반의 실시간 메시지 전달, HTTPS/WSS 보안 통신, MCP(Model Context Protocol) 연동 AI 봇 응답을 핵심 기능으로 한다.

### 1.3 핵심 기술 차별점

| 기술 | 적용 목적 | 적용 없으면 |
|------|-----------|-------------|
| Redis Pub/Sub | 채널 메시지 브로드캐스트, 다중 서버 동기화 | 단일 서버 외 확장 불가, 실시간성 저하 |
| HTTPS / WSS | 보안 WebSocket 연결 | 브라우저가 wss:// 강제 요구, 배포 불가 |
| MCP (Claude) | @AI 멘션 시 AI 응답 생성 | 일반 채팅 앱과 차별점 없음 |

### 1.4 대상 사용자

- **비회원**: 회원가입 페이지 접근 가능. 채팅 기능 전면 차단
- **회원(일반)**: 채널 참여·채팅·AI 멘션·DM 사용 가능
- **관리자**: 채널 생성·삭제·공지, 회원 관리, 전체 메시지 삭제 권한

## 2. 기술 스택 및 환경

### 2.1 기술 스택

| 구분 | 기술 | 비고 |
|------|------|------|
| Frontend | Vue.js 3 (Composition API), Vite, Pinia, Vue Router 4, Axios | |
| Backend | Spring Boot 3, Spring Security, Spring Data JPA | |
| WebSocket | STOMP over WebSocket (SockJS 폴백) | |
| Database | MySQL 8 | |
| Cache / Pub/Sub | Redis 7 | 메시지 브로드캐스트, 안읽은 메시지 수, 온라인 상태 |
| 인증 | JWT (Access Token + Refresh Token) | |
| AI 연동 | Anthropic Claude API (MCP via claude-sonnet) | @AI 멘션 응답 |
| 보안 통신 | HTTPS + WSS (Let's Encrypt 또는 self-signed for dev) | |
| 파일 스토리지 | 로컬 파일 시스템 | 이미지·파일 첨부 |

### 2.2 인증 정책

| 항목 | 정책 |
|------|------|
| Access Token 만료 | 30분 |
| Refresh Token 만료 | 7일 |
| 저장 위치 | Access: 메모리(Pinia store) / Refresh: HttpOnly Cookie |
| 자동 갱신 | Access Token 만료 시 Refresh Token으로 자동 재발급 |
| 로그아웃 | Refresh Token DB 무효화 + Cookie 삭제 |
| 소켓 인증 | STOMP CONNECT 프레임의 `Authorization` 헤더로 JWT 검증 |

> **이전 프로젝트와 다른 점**: sessionStorage 대신 Refresh Token 방식 도입. 탭을 닫아도 로그인 유지, 보안 강화.

### 2.3 파일 업로드 정책

| 항목 | 정책 |
|------|------|
| 허용 파일 | 이미지: jpg, jpeg, png, gif, webp / 일반: pdf, zip, txt, md (확장자 추가 가능) |
| 이미지 최대 크기 | 10MB |
| 일반 파일 최대 크기 | 20MB |
| 메시지당 최대 첨부 | 5개 |
| 저장 경로 | `./uploads/chat/{channelId}/` |
| 파일명 | `{UUID}-{originalName}` |

### 2.4 Redis 사용 목적별 Key 설계

| 용도 | Key 패턴 | TTL |
|------|----------|-----|
| 채널 구독 Pub/Sub | `chat:channel:{channelId}` | 없음 (채널 단위) |
| DM Pub/Sub | `chat:dm:{dmRoomId}` | 없음 |
| 온라인 상태 | `presence:{userId}` | 60초 (하트비트 갱신) |
| 안읽은 메시지 수 | `unread:{userId}:{channelId}` | 없음 |
| Refresh Token | `refresh:{userId}` | 7일 |
| AI 응답 큐 | `ai:queue:{channelId}` | 10분 |

### 2.5 URL 구조

| 경로 | 설명 |
|------|------|
| `/` | 홈 (로그인 상태면 `/channels`로 리다이렉트) |
| `/login` | 로그인 페이지 |
| `/join` | 회원가입 페이지 |
| `/channels` | 채널 목록 사이드바 + 메인 영역 (레이아웃 기준점) |
| `/channels/:channelId` | 특정 채널 채팅 화면 |
| `/dm/:dmRoomId` | 1:1 다이렉트 메시지 화면 |
| `/admin` | 관리자 대시보드 |
| `/admin/channels` | 채널 관리 |
| `/admin/users` | 회원 관리 |

## 3. 전체 레이아웃 구조

### 3.1 기본 레이아웃 (`/channels` 이후 전 페이지 공통)

```
┌─────────────────────────────────────────────────────────┐
│  [로고/홈]        ChatterAI          [알림] [내 프로필]  │  ← GNB (64px)
├──────────┬──────────────────────────────────────────────┤
│          │  채널명 + 참여자 수                 [검색]   │  ← 채널 헤더 (56px)
│  사이드바 ├──────────────────────────────────────────────┤
│  (240px) │                                              │
│  · 채널  │              메시지 영역                      │
│    목록  │          (스크롤, 무한 로드)                  │
│  · DM    │                                              │
│  · 참여자│──────────────────────────────────────────────┤
│    온라인│  메시지 입력창 + 파일 첨부 + 이모지 + 전송   │  ← 입력 영역
└──────────┴──────────────────────────────────────────────┘
```

### 3.2 사이드바 영역

| 섹션 | 내용 |
|------|------|
| 채널 섹션 헤더 | `채널` 텍스트 + `+` 버튼 (관리자만 표시) |
| 채널 목록 | 채널명, 안읽은 메시지 수 뱃지, 현재 선택 채널 강조 |
| DM 섹션 헤더 | `다이렉트 메시지` 텍스트 + `+` 버튼 |
| DM 목록 | 상대방 이름, 온라인 상태 점, 안읽은 수 뱃지 |
| 하단 내 정보 | 아바타 + 닉네임 + 온라인 상태 + 설정 아이콘 |

### 4.1 로그인 (`/login`)

#### 4.1.1 화면 구성

| 필드 | 설명 |
|------|------|
| 아이디 (텍스트) | 사용자 ID 입력 |
| 비밀번호 (패스워드) | 비밀번호 입력 |
| 로그인 버튼 | 제출 |
| 회원가입 링크 | `/join` 이동 |

#### 4.1.2 동작

- 로그인 성공: Access Token → Pinia store 저장, Refresh Token → HttpOnly Cookie, `/channels`로 이동
- 로그인 실패: 인풋 하단 에러 메시지 표시
- 이미 로그인 상태에서 접근: `/channels`로 리다이렉트

---

### 4.2 회원가입 (`/join`)

#### 4.2.1 화면 구성

| 필드 | 유형 | 필수 | 유효성 검사 |
|------|------|------|-------------|
| 아이디 | 텍스트 | 필수 | 영문+숫자, 4~20자, 중복 불가 |
| 비밀번호 | 패스워드 | 필수 | 영문+숫자+특수문자 조합, 8자 이상 |
| 비밀번호 확인 | 패스워드 | 필수 | 비밀번호 일치 여부 |
| 닉네임 | 텍스트 | 필수 | 2~20자, 중복 불가 |
| 가입 버튼 | 제출 | — | 모든 필드 유효 시 활성화 |

- 아이디·닉네임 중복 확인은 폼 제출 시 서버 검증 (실시간 체크 미지원)
- 가입 성공 시 `/login`으로 이동

---

### 4.3 채널 채팅 (`/channels/:channelId`)

#### 4.3.1 채널 입장 조건

| 사용자 | 결과 |
|--------|------|
| 비회원 | `/login`으로 리다이렉트 |
| 회원 (비참여 채널) | 자동 참여 후 입장 (공개 채널), 비공개 채널은 초대 필요 |
| 회원 (참여 채널) | 바로 입장 |
| 관리자 | 모든 채널 바로 입장 |

#### 4.3.2 메시지 영역

- 날짜 구분선: 날짜가 바뀌는 지점에 `YYYY년 MM월 DD일` 구분선 표시
- 메시지 묶음: 같은 사용자가 5분 이내 연속 전송 시 아바타/닉네임 생략 (첫 메시지에만 표시)
- 스크롤: 최하단 고정. 새 메시지 수신 시 자동 스크롤 (단, 사용자가 위로 스크롤 중이면 "N개의 새 메시지 ↓" 뱃지 표시)
- 무한 스크롤: 위로 스크롤 시 이전 메시지 20개씩 추가 로드

#### 4.3.3 메시지 아이템 구성

| 요소 | 설명 |
|------|------|
| 아바타 | 사용자 아바타 이미지 또는 이니셜 원형 |
| 닉네임 | 클릭 시 DM 시작 가능 |
| 시각 | `HH:MM` |
| 내용 | 텍스트, 이미지 인라인 미리보기, 파일 다운로드 링크 |
| AI 응답 메시지 | 아바타 `🤖 AI`, 닉네임 `ChatterAI`, 배경색 구분 |
| 호버 액션 | 이모지 반응 `😊` + 답장 `↩` + 더보기 `⋯` |
| 더보기 메뉴 | 메시지 수정 / 삭제 (본인) + 삭제 (관리자) |

#### 4.3.4 메시지 입력창

| 요소 | 설명 |
|------|------|
| 텍스트 입력 | 멀티라인, `Enter` 전송 / `Shift+Enter` 줄바꿈 |
| 파일 첨부 버튼 | 클립 아이콘, 클릭 시 파일 선택 (최대 5개) |
| 이모지 버튼 | 이모지 피커 팝업 |
| 전송 버튼 | 클릭 또는 `Enter` |
| @AI 멘션 | `@AI` 타이핑 시 메시지에 AI 멘션 표시, 전송 시 AI 응답 트리거 |

#### 4.3.5 @AI 멘션 동작

1. 사용자가 `@AI 질문 내용`을 입력 후 전송
2. 해당 메시지가 채널에 즉시 표시 (일반 메시지와 동일)
3. `AI가 답변을 생성 중입니다...` 로딩 메시지가 AI 이름으로 표시
4. Backend → MCP → Claude API 응답 수신
5. 로딩 메시지가 실제 AI 응답으로 교체되어 채널에 전송
6. AI 응답은 수정·삭제 불가 (관리자만 삭제 가능)

#### 4.3.6 이모지 반응

- 메시지 호버 시 `😊` 버튼 노출
- 클릭 시 이모지 피커 팝업 표시
- 동일 이모지 재클릭 시 반응 취소
- 메시지 하단에 이모지별 반응 수 표시 (예: `👍 3 😂 1`)

### 4.4 1:1 다이렉트 메시지 (`/dm/:dmRoomId`)

#### 4.4.1 DM 방 생성

- 사이드바 `+` 버튼 클릭 → 사용자 검색 모달 → 선택 시 DM 방 생성 또는 기존 DM 방 입장

#### 4.4.2 화면 구성

- 채널 채팅과 동일한 메시지 영역 구조
- `@AI` 멘션 지원 (DM에서도 AI 호출 가능)
- 상단 헤더: 상대방 닉네임 + 온라인 상태 점

---

### 4.5 채널 관리 (관리자 전용)

#### 4.5.1 채널 생성

- 사이드바 `+` 버튼 클릭 (관리자만 노출)
- 채널명 (필수, 2~30자, 중복 불가), 설명 (선택), 공개/비공개 설정

#### 4.5.2 채널 삭제

- 관리자 대시보드 `/admin/channels`에서 삭제
- 삭제 시 해당 채널의 모든 메시지·첨부파일 삭제 (확인 모달 필수)

#### 4.5.3 공지 메시지

- 관리자는 채널 헤더의 `📌 공지` 버튼으로 특정 메시지를 공지로 지정
- 공지 메시지는 채널 헤더 아래 고정 배너로 표시

## 6. 공통 컴포넌트 명세

### 6.1 메시지 아이템 컴포넌트 (`MessageItem`)

- `isFirstInGroup`: 같은 사용자 연속 메시지 중 첫 번째인지 여부 (아바타/닉네임 표시 여부)
- `isAiMessage`: AI 응답 메시지 여부 (배경색 구분, 수정 불가)
- `isOwn`: 본인 메시지 여부 (수정/삭제 버튼 표시)
- `hasAttachments`: 첨부파일 여부

### 6.2 메시지 입력창 컴포넌트 (`MessageInput`)

- `@mention-ai`: `@AI` 입력 감지 이벤트
- `@file-attach`: 파일 선택 이벤트
- `@submit`: 메시지 전송 이벤트

### 6.3 온라인 상태 점 (`PresenceDot`)

| 상태 | 색상 |
|------|------|
| 온라인 | 초록 (#22C55E) |
| 자리비움 (10분 이상 미활동) | 노랑 (#EAB308) |
| 오프라인 | 회색 (#6B7280) |

### 6.4 알림 토스트

- 성공 / 에러 / 정보 3종
- 우측 하단 고정, 3초 후 자동 사라짐
- 동시 최대 3개 쌓임

### 6.5 사용자 검색 모달 (DM 시작용)

- 닉네임 검색 (최소 1자)
- 검색 결과 목록에 온라인 상태 표시
- 선택 시 DM 방 생성 또는 기존 방 이동

## 7. AI 멘션 상세 스펙

### 7.1 트리거 조건

- 메시지 내 `@AI` 문자열 포함 시 AI 응답 트리거
- 메시지에 `@AI`가 없으면 일반 채팅 메시지로만 처리

### 7.2 AI 응답 맥락 (Context)

- 해당 채널의 최근 메시지 10개를 MCP 컨텍스트로 전달
- DM의 경우 해당 DM 방의 최근 메시지 10개 전달
- AI에 전달되는 시스템 프롬프트: `"당신은 ChatterAI의 AI 어시스턴트입니다. 채팅 맥락을 바탕으로 도움이 되는 답변을 한국어로 제공하세요."`

### 7.3 AI 멘션 제한 (일반 회원)

| 항목 | 제한 |
|------|------|
| 일일 사용 횟수 | 20회 (자정 기준 초기화) |
| 초과 시 처리 | 토스트 메시지: `"오늘의 AI 사용 횟수(20회)를 모두 사용했습니다."` |
| 제한 카운트 저장 | Redis: `ai:usage:{userId}:{date}` TTL 24h |

### 7.4 AI 응답 실패 처리

- Claude API 타임아웃 (10초 초과): 로딩 메시지 → `"AI 응답에 실패했습니다. 잠시 후 다시 시도해 주세요."` 메시지로 교체
- 에러 메시지도 채널에 전송 (AI 봇 이름으로)

## 8. 실시간 이벤트 명세 (STOMP)

### 8.1 구독 경로 (Subscribe)

| 경로 | 설명 |
|------|------|
| `/topic/channel/{channelId}` | 채널 메시지 수신 |
| `/topic/dm/{dmRoomId}` | DM 메시지 수신 |
| `/topic/channel/{channelId}/typing` | 타이핑 인디케이터 수신 |
| `/user/queue/notification` | 개인 알림 수신 (안읽은 수 갱신 등) |

### 8.2 발행 경로 (Publish)

| 경로 | 설명 |
|------|------|
| `/app/channel/{channelId}/send` | 채널 메시지 전송 |
| `/app/dm/{dmRoomId}/send` | DM 메시지 전송 |
| `/app/channel/{channelId}/typing` | 타이핑 상태 전송 |

### 8.3 타이핑 인디케이터

- 입력창에 타이핑 시 `/app/channel/{channelId}/typing` 발행
- 채널 메시지 입력창 하단에 `OOO님이 입력 중...` 표시
- 3초 동안 추가 타이핑 없으면 자동 사라짐

## 9. 비기능 요구사항

| 항목 | 요구사항 |
|------|----------|
| 응답 속도 | 메시지 전송 후 수신까지 200ms 이하 (로컬 기준) |
| 동시 접속 | 최소 100명 동시 WebSocket 연결 처리 |
| 보안 | HTTPS/WSS 강제, JWT 만료 검증, XSS 방지 (HTML 이스케이프) |
| 가용성 | Redis 장애 시 폴백 없이 에러 반환 (MVP 범위 외) |
| 로깅 | 메시지 전송/수신, AI 멘션, 에러 모두 서버 로그 기록 |

## 10. 개발 범위 (MVP)

### 포함

- 채널 기반 실시간 채팅 (WebSocket/STOMP)
- Redis Pub/Sub 메시지 브로드캐스트
- @AI 멘션 → Claude API 응답
- 1:1 DM
- 파일·이미지 첨부
- 이모지 반응
- 타이핑 인디케이터
- 온라인 상태 표시
- JWT (Access + Refresh Token)
- 관리자 기능 (채널 관리, 회원 관리)
- HTTPS/WSS 설정 (개발: self-signed, 배포: Let's Encrypt)

### 제외 (향후 확장)

- 메시지 검색 (Elasticsearch)
- 음성/화상 통화
- 메시지 스레드 (답장 체인)
- 채널 초대 코드
- 이메일 인증
- 소셜 로그인 (OAuth2)
- 푸시 알림 (FCM)
- 모바일 앱
